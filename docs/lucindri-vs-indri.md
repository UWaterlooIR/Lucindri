# Lucindri vs. C++ Indri — query-language differences catalog (TASK-0005)

Behavioral conformance comparison of **Lucindri** against the reference **C++ Indri 5.21**
(`/ssd-8TB/installs/indri-5.21/`) on the query-language elements Lucindri claims to implement
(repo `README.md`). Generated by `scripts/diff_indri.sh` (reproducible). This is a
**catalog-and-triage** document; fixes are separate tasks.

## Method

The **same logical documents** (a `docid` + a **markup-free** text body — Lucindri's actual model)
are indexed by each engine in **its native format**:
- **Lucindri:** JSONL via the `climbmix` parser (`docid`→`externalId`, `contents`→`fulltext`) — its
  real target format, giving clean ids and clean bodies (no markup tokens).
- **Indri:** trectext via `IndriBuildIndex` (`DOCNO`/`TEXT`).

Analysis aligned (Indri **Krovetz** ≈ Lucindri **KStem**, both lowercase); probe tokens are
stem-invariant + non-stopword so differences are query **semantics**, not tokenization. μ = 2000
Dirichlet in both. For each probe, compare the **retrieved doc set** (primary correctness signal),
then the **ranked order**. Where Lucindri renames an Indri operator, the equivalent Indri form is
used (`#scoreif`↔`#filreq`, `#scoreifnot`↔`#filrej`).

Corpus: `1:apple cat dog`, `2:banana cat`, `3:apple sun moon`, `4:cat dog moon`, `5:dog apple`,
`6:tree bird fish`.

**Severity:** **Critical** = wrong/ignored → changes which docs are retrieved. **Major** = retrieved
set diverges meaningfully. **Minor** = benign (tie-breaking, unprobed analysis).

### Fixed in TASK-0006 (were Critical / Major)

Both former divergences now conform to Indri (doc sets match; only benign tie-breaking remains):
1. **`#scoreif` / `#scoreifnot` now filter.** Added native `IndriFilterRequireQuery` /
   `IndriFilterRejectQuery` (Indri `#filreq`/`#filrej`): the scored subquery drives the candidate
   set, kept (require) or dropped (reject) by whether the filter matches. `#scoreif(dog #combine(cat))`
   → {1,4}; `#scoreifnot(dog #combine(cat))` → {2} (rank-matches Indri). (Was: both returned
   {1,2,4,5} — the parser marked the clause `MUST`/`MUST_NOT` but `IndriAndQuery` ignored it.)
2. **`#not` no longer leaks.** `IndriNotScorer.iterator()` returned the negated term's postings;
   it now returns an empty iterator, so `#not` only modifies belief over docs that are candidates
   via positive siblings. `#combine(cat #not(dog))` → {1,2,4}. (Was: {1,2,4,5}, retrieving doc 5
   which has `dog` but no `cat`.)

### Conformant (doc sets match Indri)

`term`, `#combine`, `#or`, `#wand`, `#wsum`, `#max`, `#band`, `#N`, `#uwN`, `#syn`, and
`#band(#syn …)` all retrieve **the same documents as Indri**. In particular `#band(#syn …)` = {1,2,5}
matches Indri — confirming the **TASK-0002** proximity/compound-operand fixes conform to the
reference.

## Category C — scoring / ranking

With clean bodies on both sides, **Lucindri's Dirichlet scores match Indri's to ~5 decimals.**
Example (`cat`, μ=2000): Indri `doc2=-1.67231, doc4=-1.67281, doc1=-1.67281`; Lucindri
`doc2=-1.6723129, doc1=-1.6728122, doc4=-1.6728122`. This confirms Lucindri's scoring conforms to
the reference (validating **TASK-0003** against Indri itself).

**Rank differences are tie-breaking only.** Every `rank-DIFFER` above is a reordering among
documents with **identical scores** (e.g. doc1 and doc4 tie at the same value in each engine); the
two engines break ties by internal doc id in different orders. Where scores genuinely differ, the
rankings **match** (`#1`, `#combine(#or …)` rank-match exactly). **Severity: Minor / benign.**

## Category B — analysis / tokenization (not stressed here)

- **KStem vs Krovetz:** probe tokens were chosen stem-invariant to isolate operator semantics, so
  stemming was not exercised. Known Lucindri KStem quirk (TASK-0004): KStem leaves some plurals
  unstemmed (`dogs`, `mice`), so `dog`≠`dogs` at query time; Krovetz may differ. **TODO:** a
  per-word stemming probe in both engines is a future pass.
- **Query stopwords:** Lucindri's query analyzer **always** removes English stopwords (hardcoded);
  Indri removes them only with a configured `<stopper>`. Diverges for stopword-containing queries
  unless Indri is given the same list. (Probes above are non-stopword.)

## Notes

- Lucindri is compared in **JSONL** (its target format). An earlier attempt using *trectext* for
  Lucindri hit parser bugs unrelated to query semantics (NPE on empty `fieldNames`, `<DOCNO>`
  extracted only when it preceded `<DOC>`, markup tokens in `fulltext`). **Those are fixed in
  TASK-0007** — `trectext` now extracts standard `<DOCNO>`, indexes a markup-free body from
  configurable content tags, and is null-safe — so a trectext-for-Lucindri comparison would now
  agree with Indri on ids/doc sets.
